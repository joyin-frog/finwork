# Spec: 知识库页面布局 v3

> 版本 v1.0 / 2026-06-08
> 目标：搜索主导 + 浏览/预览侧栏 + 上传收纳为 FAB，去除 tab 切换。

---

## 1. 现状

`app/knowledge/page.tsx` 有两个 tab：「浏览」和「搜索」。浏览 tab 里有上传区（drag-drop 占 ~120px）、分类筛选、文档列表；搜索 tab 里有 rg 搜索栏和结果列表。右上角有 tab 切换按钮。

问题：
- 搜索入口不够直接，藏在一个 tab 里
- 上传区占空间大，日常使用频率低却常驻
- 「浏览」和「搜索」两套逻辑同页却割裂

## 2. 目标

| 区域 | 现状 | 目标 |
|------|------|------|
| 搜索栏 | 在搜索 tab 内 | 移到页面最上方，始终可见，替代顶部搜索 tab 按钮 |
| 文档列表 | 在浏览 tab 内 | 和搜索结果合并为一个列表；每条右侧加「查看原文」按钮 |
| 上传 | drag-drop 区域，~120px 高 | 缩小为右下角 FAB 按钮，点击弹出上传对话框 |
| 分类筛选 | category chips 在浏览 tab 内 | 保留在搜索栏下方，始终可见 |
| 预览 | 右侧侧栏 | 保持不变 |
| Tab 切换 | 浏览/搜索两个 tab | 删除 |

## 3. 布局

```
┌─────────────────────────────────────────────────┬──────────────┐
│ [输入框: 搜索知识库...] [正则]                       │              │
│ [报销制度] [合同] [财务规范] [税务] [通用]            │   预览       │
├─────────────────────────────────────────────────┤   (侧栏)     │
│ 文档列表 / 搜索结果                                │              │
│  ┌─────────────────────────────────────────────┐ │              │
│  │ 文件名.pdf              [查看原文]             │ │              │
│  │ category · 命中 12 次                        │ │              │
│  │ L1: ...匹配...                               │ │              │
│  │ L33: ...匹配...                              │ │              │
│  │ ...                                         │ │              │
│  └─────────────────────────────────────────────┘ │              │
│                                                  │              │
│  [+] (FAB, 右下角)                                │              │
└─────────────────────────────────────────────────┴──────────────┘
```

### 3.1 搜索栏（页面顶部，始终可见）

- 替换当前 `.codex-topbar` 里的「搜索」tab 按钮
- 搜索栏 = 输入框 + 正则 checkbox + 回车/按钮搜索
- 清空搜索框时自动恢复为文档列表（无搜索结果的浏览模式）
- `onChange` 不带搜索，`onKeyDown Enter` 或点搜索按钮才触发 rg 搜索

### 3.2 分类筛选 chips

- 搜索栏下方，保持现有 `.knowledge-category-chips` 样式和逻辑
- 浏览模式（无搜索 query）：筛选文档列表
- 搜索模式（有结果）：不过滤（rg 返回的已是全量），可以隐藏或灰掉

### 3.3 文档列表 / 搜索结果

- 浏览模式：显示 `filteredDocs`，每个 item 右侧有「查看原文」按钮
- 搜索模式：显示 `results.files`，每个 item 下方有匹配片段（可滚动），右侧有「查看原文」
- 「查看原文」点击 → 右侧侧栏打开，用 `FilePreviewPage` 展示原文件（浏览模式）或文本镜像预览（搜索模式，带命中行高亮和导航）
- 搜索结果卡片内的匹配片段列表：点击跳转预览 + 设焦点行（保持 v2 的 hitLines/hitIndex 导航逻辑）

### 3.4 上传 FAB

- 右下角固定定位，圆形按钮，`+` 图标
- 点击弹出模态框或 popover：
  - 文件选择 input
  - 分类下拉（自动检测 / 手动选）
  - 「上传并索引」按钮
  - 进度提示
- 上传完成后刷新文档列表，关闭弹窗
- FAB 在侧栏展开时不与侧栏重叠

### 3.5 预览侧栏

- 保持现有逻辑（可拖拽调整宽度，可折叠）
- 浏览模式下的「查看原文」：用 `FilePreviewPage`（现有共用组件）
- 搜索模式下的「查看原文」：用文本镜像预览 + 导航 bar（保持 v2 的 hitLines/hitIndex 逻辑）

## 4. 交互

| 操作 | 行为 |
|------|------|
| 输入关键词 + Enter / 点搜索 | 调 rg 搜索 API，列表切为搜索结果 |
| 清空搜索框 | 列表切回文档列表（浏览模式） |
| 点某个文档的「查看原文」 | 打开右侧侧栏，`FilePreviewPage` 展示原文件 |
| 点某个搜索结果的匹配片段 | 打开右侧侧栏，文本镜像预览 + 命中行高亮，焦点跳转到该行 |
| 点某个搜索结果的「查看原文」 | 打开右侧侧栏，文本镜像预览 + 命中行高亮 + ↑↓ 导航 |
| ↑↓ 键盘 | 搜索预览内切换命中行（保持 v2） |
| 侧栏内 ↑↓ 按钮 | 同上，循环 |
| 点右下角 FAB | 弹出上传弹窗 |
| 上传完成 | 关闭弹窗，刷新文档列表 |

## 5. 状态管理

- 去掉 `tab` 状态（不再需要）
- 保留 `query`，清空 `query` = 浏览模式，有 `query` + `results` = 搜索模式
- `previewMode` 状态：`"file"`（浏览模式查看原文）或 `"search"`（搜索模式查看结果），决定右侧预览用哪个组件
- 保留 v2 的 `hitLines` / `hitIndex` / `previewDocId` 导航状态
