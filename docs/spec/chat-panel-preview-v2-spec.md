# 聊天文件面板与右侧预览页 V2 Spec

## 范围

本次只调整聊天页文件面板、右侧预览栏、独立预览页以及相关左侧导航激活态样式。

不在本次范围内：

- 文件格式支持范围变更。
- 对话数据模型变更。
- 预览解析链路重构。

## 用户需求拆解

### 1. 左侧导航激活态

- `置顶`、`最近`、左侧当前激活导航项恢复可见的激活态。
- 激活态和 hover 态都应显示边框与背景。
- 非激活且未 hover 时保持透明。

### 2. 顶部按钮激活态

- `面板` 按钮在面板打开时显示激活态背景和边框。
- `右侧栏` 按钮在右侧预览栏展开时显示激活态背景和边框。
- 如果用户从消息区点击文件触发右侧预览，则右侧栏按钮进入激活态。
- 面板和右侧栏允许同时处于激活态。

### 3. 面板默认打开规则

- 当前对话没有任何文件时，面板默认关闭。
- 当前对话存在文件时，面板默认打开，右侧栏默认关闭。
- 当 assistant 新增输出文件时，面板自动展开一次，便于发现新文件。
- 自动展开输出文件面板时，不主动关闭右侧栏。

### 4. 面板与右侧栏交互

- 取消互斥逻辑。
- 点击右侧栏按钮时，只切换右侧栏，不影响面板当前状态。
- 点击面板按钮时，只切换面板，不影响右侧栏当前状态。
- 点击消息区文件时，打开右侧栏并切换预览内容，不强制关闭面板。
- 点击面板文件时，打开右侧栏并切换预览内容，可保持面板继续展开。

### 5. 面板浮层定位

- 面板浮层始终贴齐中间对话区右边缘，而不是贴齐顶部按钮本身右边缘。
- 当右侧栏展开或拖动分栏比例时，面板浮层位置随中间对话区宽度同步移动。
- 面板浮层相对聊天页顶部按钮向下展开，但右边界对齐中间对话区。

### 6. 右侧栏默认宽度

- 用户首次打开右侧栏时，中间对话区与右侧栏默认占比为 `1:1`。
- 用户拖动过分栏后，后续保持用户当前拖动结果。

### 7. 预览页外观

- 预览页主内容与外层容器之间不保留额外留白。
- 预览页外层不使用圆角。
- 右侧栏内预览页铺满侧栏内容区域。

### 8. 预览页头部

- 文件名与 `打开方式` 位于同一行。
- 两者位于同一个头部卡片内。
- 文件名与 `打开方式` 文本样式保持一致的视觉等级。
- 没有当前文件时，头部卡片保留 `打开文件` 入口。

### 9. 面板文件行样式

- 面板内单个文件行采用单层卡片，不出现外层套内层的重复盒子感。
- 文件名与 `打开方式` 位于同一行、同一卡片内。
- 文件名与 `打开方式` 文本样式保持一致的视觉等级。

### 10. 面板透明度

- 面板浮层不再使用明显透明玻璃效果。
- 面板背景改为实色或接近实色，降低透视感。

## 实现约束

- 继续复用现有 `ChatFilePanel`、`ChatPreviewSidebar`、`FilePreviewPage`、`OpenableFileRow`。
- 不修改文件解析逻辑，只改状态管理、结构和样式。
- 若新增状态逻辑，优先抽为纯函数，便于测试。

## 测试用例

### 自动化测试

1. 聊天页状态逻辑：
   - 无文件时默认 `filePanelOpen = false`
   - 有文件时默认 `filePanelOpen = true`
   - 新输出文件到达时面板自动展开，但不强制关闭右侧栏
   - 点击消息区或面板文件后右侧栏展开，且面板状态不被强制改写
   - 右侧栏与面板可同时打开

2. 聊天页结构：
   - `ChatFilePanel` 接收右侧栏展开态和面板右偏移
   - 顶部按钮存在激活态 class

3. 预览页结构：
   - 头部使用同一卡片承载文件名与 `打开方式`
   - 空状态时保留 `打开文件`

4. 样式约束：
   - 左侧导航 active 样式存在背景与边框
   - `.panel-tab.active` 与 `.sidebar-toggle.active` 存在激活态样式
   - `.preview-page-shell` 无圆角、无内边距
   - `.preview-sidebar` 无额外内边距
   - `.panel-popover` 使用不透明背景
   - `.panel-popover` 使用基于右侧栏宽度的对齐偏移

### 手工验收

1. 打开 `/chat/new`
   - 无文件时，面板默认关闭，右侧栏默认关闭
2. 打开带附件的历史对话
   - 面板默认打开，右侧栏默认关闭
3. 点击右侧栏按钮
   - 右侧栏打开，面板若已打开则保持打开
4. 点击面板按钮
   - 面板打开或关闭，不影响右侧栏
5. 点击消息区文件或面板文件
   - 右侧栏切换到对应预览，右侧栏按钮显示激活态
6. 拖动中间与右侧分栏
   - 面板浮层右边界随中间对话区右边缘移动
7. 查看左侧 `置顶` / `最近`
   - 激活项和 hover 项都具有背景和边框
8. 查看右侧预览页
   - 预览页无圆角、无外层缝隙
   - 文件名与 `打开方式` 同行同卡片
9. 查看面板文件行
   - 文件名与 `打开方式` 同行同卡片
10. 查看面板背景
   - 不再透出明显底层内容

## 执行步骤

1. 写入 spec 与自动化测试目标。
2. 调整聊天页状态逻辑与默认分栏宽度。
3. 调整面板和预览页结构。
4. 调整导航、面板、预览页样式。
5. 补充自动化测试。
6. 执行 `rtk npx tsc --noEmit`、`rtk npm test`、`rtk npm run build`。
7. 启动本地页面并用内置浏览器做交互验收。

- 十条用户需求全部有明确实现映射。
- 自动化测试、类型检查、构建全部通过。
- 浏览器内实际交互与 spec 一致，无明显样式回退。
