# 设置页 UI 精简补充 Spec

> 范围：基于 `docs/spec-ui-refresh.md` 的现有视觉方向，只收敛设置中心里的通用、外观、知识库控件，不扩展到整站重构。

## 目标

设置页应保持清新、克制、低噪声的项目视觉语言。用户调整外观时不需要理解“草稿 / 预览 / 保存”三段流程，控件本身应直接表达当前值，并在变更后立即生效。

## Phase 0：通用设置去重

- 通用偏好里删除“主题”选择项。
- 设置中心侧栏的通用说明不再显示主题状态，避免用户误以为主题仍在通用页配置。

验收：

- `GeneralSettings` 不再接收 `theme` / `onThemeChange`。
- 通用偏好只保留语言、货币单位、数字展示。

## Phase 1：外观设置简化

- 删除右侧预览面板。
- 删除底部“还原、取消、保存”按钮。
- 删除每个字段里的“还原默认”。
- 删除颜色圆形预设按钮。
- 颜色字段采用左文字、右颜色框布局；颜色框填满当前颜色，框内显示白色色号，点击打开系统色盘。
- 字体、字号、对比度采用左文字、右输入或选择控件布局。
- 改动后立即更新当前页面主题，并立即持久化到本地设置接口。

验收：

- 外观页没有 `appearance-preview-panel`。
- 外观页没有 `settings-actions`。
- 颜色字段没有 preset palette。
- 任意外观字段变更会调用 `/api/settings/appearance` 的 `PUT`。

## Phase 2：知识库控件适配

- “全部分类”下拉框使用设置页统一控件样式。
- “语义检索”按钮使用设置页主按钮样式，圆角、字体、颜色与项目一致。
- 手动分类相关 checkbox/select 不使用浏览器默认裸样式。

验收：

- 文档列表分类筛选和搜索分类筛选都带有知识库控件类名。
- 搜索按钮带有知识库主操作按钮类名。
- 手动分类行里的开关与下拉框有一致圆角、边框和 focus 状态。

## Phase 3：SQLite 知识库维度防护

旧知识库 embedding 维度与当前 `knowledge_vec` 表维度不一致时，不应导致聊天页初始化失败。

验收：

- reset 脚本使用与运行时代码一致的默认数据路径。
- `migrateChunksToVec` 遇到不兼容维度时跳过不兼容 chunk，而不是让 `/api/chat/recent` 初始化失败。

## Phase 4：验证

- 运行 `npm test`。
- 运行 `npm run build`。
- 启动本地 dev 服务，打开并验证 `http://localhost:3000/chat/recent?id=14`。
