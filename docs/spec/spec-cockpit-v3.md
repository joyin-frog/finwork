## 0. 目标与非目标

**目标**：

1. 删 MetricStrip（四格指标条）——含恢复条件；
2. 删 ComplianceStrip（周期性合规条）——内容三处安置；
3. 总览的团队摘要卡更名「智能体」、行点击展开该角色最近任务；
4. 新增一级空间 **「智能体」（/agents）**：花名册 + 权限透明 + 调度台账 + 有限配置；
5. 侧栏收窄：新对话（保留，双入口决策）/ 总览 / 智能体 / 资料 + 最近列表 + 设置；「技能」项移除；
6. 技能的**展示**归角色详情，**管理**留设置的技能中心。

**非目标**：

- 对话浮窗、角色动态条、派活入口退役——`spec-chat-dock.md` 范围，**本期派活入口保留**；
- 角色 rolePrompt / 工具白名单的用户编辑——**永不**（可审计性三硬性理由）；
- 角色独立聊天页——不做，角色不是对话对象，是职能域的窗口。

**命名备忘**：空间名「智能体」为用户定名（2026-07-02）。页内角色一律用财务岗位名（记账专员等）。若用户测试反馈术语困惑，备选名「团队」——记录在案，不重开讨论。

## 1. 删除项

## 3. 「智能体」空间（/agents）

**定位红线（写死）：档案馆 + 工作台账 = 多智能体的信任验证层，不是控制台。** 执行永远发生在对话里。

### 3.1 花名册

`ROLE_REGISTRY` 全员渲染（含 `available: false` 与用户停用的，样式弱化标「尚未启用」/「已停用」），每角色：

- 名称、职能域、charter；
- **数据权限域白话说明**：直接渲染 `role.dataScope`（如「payroll_records（全产品唯一有工资明细权限的角色）」）——权限透明是小团队信任的核心卖点；
- 挂载技能清单：`role.skills` 关联 skills-store 的名称与描述展示（只读；技能启停管理仍在设置的技能中心）。

**单一事实源守卫**：页面不得硬编码任何角色名/职责文案，全部来自 ROLE_REGISTRY（服务端注入）与 skills-store。

### 3.2 台账

- 每角色调度史（分页，`subagent_dispatches` 按 role_id），行：label、summary 首行、状态、时间，链接原对话；
- 「停在确认门」记录单独前置显示（blocked_reason 非空）；
- 记账专员详情附发票台账计数（承接 §1.2）。

### 3.3 有限配置

- **角色启停**：用户侧覆盖，存 `app_settings` key `agent_disabled_roles`（JSON 数组）；生效规则：spawn 枚举与派发校验 = `available && !userDisabled`；**启停动作写一行 `audit_logs`**（角色可用面的变化可审计，红线 8）；
- 不提供任何 prompt / 工具白名单编辑入口。

### 3.4 派活与对话模型（「一事一卷宗」规则，2026-07-02 定）

- 每角色「派活」按钮 → `/chat/new?prompt=让{角色名}…` 预填。不在本页面造执行通道。
- **对话模型判据：新业务事件开新卷宗，未结事项回原卷宗。** 派活永远开新对话（周期性任务每期一个卷宗：标题即任务名、列表即台账、审计回看一事一档）；「继续未完的事」的入口是关注区 gate 卡与台账的「回到原对话」，不在派活按钮上加选择菜单。
- 依据：跨期连续性靠数据层（如个税累计接力用 confirmed 库表，不用聊天记忆），角色无状态——新对话零业务损失。

### 3.5 预留：角色偏好区（本期不实现，口子记录在案）

为未来适配性预留角色详情内的「偏好」分组，边界如下：

- **内容 = 该职能域参数的按域投影**：聚合展示既有三处配置（设置页的税率/报销限额、memory.md 工作约定、profile.json 画像）中属于本域的条目；修改跳转/复用各自既有的家与确认流，**不新造配置系统**；
- **可预见的第一个住户**：开屏巡检落地后的「角色巡检开关与巡检项」；
- **红线不松动**：偏好永远是参数级（数字/开关/口径定义，结构化+确认流+审计落点）；rolePrompt 与工具白名单仍只走版本库。

### API

- `GET /api/agents`：花名册（registry 注入）+ 每角色 dispatch 汇总 + 启停状态 + 发票计数（记账）；
- `GET /api/agents/dispatches?roleId=&limit=&offset=`：台账分页；
- `POST /api/agents/toggle`：启停（写 app_settings + audit_logs）。

## 6. 验收标准

1. `metric-strip.tsx`、`compliance-strip.tsx` 删除，全仓 grep 无引用残留；
2. /agents 花名册恰好渲染 ROLE_REGISTRY 全员（含未启用态样式），角色名/职责无页面硬编码（静态守卫）；
3. 停用角色后：spawn 枚举收缩、runSubagent 对该角色返回失败（行为测试）、audit_logs 落行；重新启用恢复；
4. `/skills` 重定向生效；侧栏无「技能」项；
5. 智能体摘要卡展开返回 ≤5 条且链接正确；
6. 全套件绿 + typecheck 零错误。
