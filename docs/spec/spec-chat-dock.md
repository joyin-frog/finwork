# 对话浮窗与角色动态条 Spec

> 版本 v1.0 / 2026-07-02（设计定稿，排期在 spec-cockpit-v3 之后）
> 前置：spec-cockpit-v3（智能体空间与减法先行）；现有 chat 页的 SSE 流 / 确认门弹窗 / 多问答面板 / 文件预览体系
> 决策来源：用户提案（右下角对话浮窗 + 顶栏消息滚动条），经评估与设计文档「对话全局可触达 + 全局悬浮提问」定稿形态吻合。

---

## 0. 目标与非目标

**目标**：对话从「一个页面」变成「全局能力」——右下角浮窗，小窗 ⇄ 全屏无缝切换；总览派活入口退役，其位置换成角色工作动态条。

**非目标**：不替代全屏对话页（深工作形态不变）；不做多浮窗/多会话并行小窗；动态条不做横滚跑马灯。

**双入口决策记录（2026-07-02 用户拍板）**：侧栏「新对话」项**保留**，Mod+N 不变——浮窗与导航双入口并存。

---

## 1. 浮窗·阶段一（快速派活）

- 右下角常驻圆钮（全局，所有空间可见）；点击展开小窗（约 400×560）：输入框 + 当前会话的轻量流式回显（文本与简单工具步）；
- 「放大」按钮 → 跳 `/chat/recent?id=` **同会话全屏继续**（会话状态经现有持久化，无需额外同步机制）；
- **复杂交互自动升全屏清单**（触发即自动展开全屏并携会话）：AskUserQuestion / 多问答面板、**高风险确认门**、附件上传、文件/表格预览渲染；
- **安全红线（不可让步）**：高风险确认门绝不在小窗内渲染——不许把「确认 12 人工资生效」压成小窗里的一个小按钮。遇确认即升全屏，红线 5 的交互面不随窗口尺寸降级；
- 小窗新建会话与历史会话续聊均支持（默认新会话；从关注卡/台账进入时携目标会话）。

## 2. 浮窗·阶段二（完整嵌入）

工具卡渲染、附件、文件预览进小窗——依赖把 chat-page 的时间线组件抽成可嵌入形态，中型工程，独立排期；阶段一不阻塞。

---

## 3. 角色工作动态条（总览，接替派活入口的位置）

数据 = `subagent_dispatches` 流（进行中 + 最近完成），如「薪税专员 10:24 完成 6 月试算草稿」「记账专员正在核对报销批次…」。

**三条克制规则（写死）**：

1. **轮换不横滚**：几秒淡入换一条，不做弹幕/marquee；
2. **无活动不装忙**：无近期派发时显示当前日历窗口的一句话（来自 cockpit-suggestions），绝不编造动态；
3. **每条可点进原会话**：可追溯，不是装饰。

**依赖顺序**：动态条仅在浮窗阶段一上线、派活入口退役后接管该位置——总览不能出现没有对话入口的空窗期。

## 5. 开放问题（不阻塞 D1）

1. 小窗尺寸/位置记忆（localStorage）；
2. 浮窗在打包桌面端多显示器下的定位；
3. 动态条与「最近工作」卡的信息重叠度——观察使用后决定动态条是否只显「进行中」。
