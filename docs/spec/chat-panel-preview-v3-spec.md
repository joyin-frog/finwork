# 聊天文件预览与输入交互 V3 Spec

## 范围

本次增量只覆盖以下 8 条需求：

1. 面板文件取消 `打开方式`，点击直接进入预览。
2. 面板文件行去掉背景和边框，缩小上下间距。
3. 聊天页所有文件入口都可直接进入右侧预览。
4. `chat/recent?id=14` 这类页面中，左侧当前对话标题行显示激活态背景和边框。
5. 输入框在输入文字或添加附件时自动增高。
6. 上传按钮 hover 反馈增强，提升可点击性。
7. 发送按钮默认带背景，支持点击和 Enter 发送。
8. 聊天滚动离开底部时，在输入框上方显示回到底部按钮。

## 用户需求拆解

### 1. 面板文件交互

- 面板内文件不再显示 `打开方式`。
- 面板内文件点击后直接切换右侧预览。
- 面板文件保持轻量列表样式，不显示额外边框或背景块。
- 相邻文件的上下间距收紧。

### 2. 所有文件入口统一预览

以下入口都必须打开右侧预览，而不是只显示文本或弹图片层：

- 用户消息气泡顶部的文件 pill。
- assistant 输出区下方的文件列表。
- 面板内文件列表。
- Markdown content 中自动识别或显式链接的文件。
- 输入区中尚未发送的上传附件。
- 输入区中引用的历史文件。
- 消息中的图片缩略图。

### 3. 草稿附件预览模型

- 未发送附件不属于已持久化会话文件，需要走草稿预览模型。
- 草稿预览至少支持：图片、Markdown、TXT、PDF、Excel、DOCX。
- 草稿预览数据来源于内存中的 `dataUrl` / `text`，不依赖本地路径或会话文件路径。

### 4. 左侧最近对话激活态

- `chat/recent?id=<id>` 页面中，左侧对应会话标题行应显示激活态。
- 激活态表现和导航 active 一致：有背景、有边框。
- `最近` 分组按钮保持其自身展开态，不替代具体对话行的激活态。

### 5. 输入区高度自适应

- 输入区容器不能写死为固定高度。
- 输入文字导致 textarea 变高时，容器同步变高。
- 上传或引用附件导致 tray 占用更多空间时，输入区整体同步增高。

### 6. 按钮可点击性

- 上传按钮 hover 时应有更明确的背景反馈。
- 上传按钮点击热区应足够大。
- 发送按钮在可用状态下默认有背景。
- Enter 发送保持可用，Shift+Enter 仍然换行。

### 7. 回到底部按钮

- 当消息列表滚动离开底部阈值时，输入框上方展示 `向下箭头` 按钮。
- 点击该按钮后平滑滚动到消息底部。
- 处于底部时按钮隐藏。
- 若用户仍在底部，新增消息时继续自动跟随到底部。
- 若用户已经滚离底部，新增消息时不强制打断当前位置。

## 自动化测试

1. 预览选择逻辑：
   - 会话文件映射为 `conversation` 预览选择
   - 未发送附件映射为 `draft` 预览选择
   - `displayFile` 可根据 `storagePath / dataUrl` 自动分流
   - 滚动阈值函数可判断是否展示回底按钮

2. 聊天页结构：
   - `FileTray` 支持未发送附件和引用文件点击预览
   - `UserBubble` / `MarkdownMessage` / 图片消息都调用统一预览逻辑
   - 聊天页存在 `scroll-to-bottom-button`

3. 导航结构：
   - `AppNav` 会根据当前 `id` 给最近对话行附加 active class

4. 样式约束：
   - 面板文件行不再渲染 `open-with-button`
   - `.panel-popover .openable-file-row.compact` 无边框、无背景
   - `.recent-item.active` 或 `.recent-item-row.active` 存在边框和背景
   - `.codex-composer` 不再固定高度
   - `.composer-send-button` 默认有背景
   - `.scroll-to-bottom-button` 样式存在

## 手工验收

1. 打开 `/chat/recent?id=14`
   - 左侧对应对话标题行有激活态背景和边框
2. 打开带文件历史对话
   - 面板文件仅点击即可进入右侧预览
   - 面板文件无 `打开方式`
   - 面板文件行无额外边框背景，间距更紧
3. 点击消息区文件
   - 输入文件、输出文件、content 文件都能进入右侧预览
4. 添加但不发送一个附件
   - 输入区附件 chip 可点击进入右侧预览
5. 输入多行文字或连续添加多个附件
   - 输入区高度自动增长，无固定截断
6. hover 上传按钮
   - 背景反馈明显，点击不费劲
7. 发送按钮可用时
   - 默认带背景，点击和 Enter 都能发送
8. 滚动消息到上方
   - 输入框上方出现回底按钮
   - 点击后平滑回到底部并隐藏按钮
