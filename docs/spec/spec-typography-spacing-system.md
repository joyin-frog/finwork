# Spec: 统一排版与间距设计语言(Typography + Spacing System)

> 状态:待审阅(草案)
> 日期:2026-06-22
> 范围:把全仓散落的字号/字重/行距/字间距/间距,收敛成两套语义 token——**字阶(重要性分层)+ 间距阶(关系分层)**,建立统一设计语言,治理"廉价 AI 感"的结构性根因(排版无层级、间距无节奏)。**纯视觉,无逻辑/数据变更。**

---

## 决策记录(2026-06-22 已确认)

- **① 字阶取值:采纳本 spec**。新增 `text-display`=24;**卡片标题 `text-title` 16/600**(核心层级招);**正文锁 14 不放大**(保密度)。
- **② 间距:两项都做**,重心在**补 24/32 章节间距**(呼吸来源);砍 6/10 半步作机械迁移(6→8 或 4、10→8 或 12),极紧处可例外退 4。
- **③ `text-overline` 签名层:本版不做**。大写 + 字间距是拉丁玩法,本应用以中文为主、不适用;签名元素改走中英通用方案(分组标题前小色条 / 圆点)**另开探索**,不阻塞本系统。
- **实施顺序**:Phase 0 token → 1 shadcn 原语 → 2 总览 → 3 对话 → 4 侧栏 → 5 其余 → 6 收尾。

---

## 背景与诊断(量化)

- **字号扁平**:全仓 `text-xs`(12)206 处、`text-sm`(14)94 处,16px 以上**仅 2 处**;另有 `text-[10px]/[11px]/[0.625rem]/[13px]/[15px]` 等一次性散值——既扁平又无系统。
- **字重单一**:`font-medium`(500)85 处一统,`font-normal` 26、`font-semibold` 10,几乎无对比。
- **行距/字间距**:`leading-*` 仅约 15 处、`tracking-*` 约 12 处,基本吃默认。
- **间距**:`gap` 挤在 ≤16px,`gap-1.5`(6)65 处、`gap-2.5`(10)16 处是**off-grid 半步污染**;`gap-6`(24)仅 2 处,24 以上几乎为零 → 无章节/大区分隔,满屏一坨。
- **既有地基(收编,不另起)**:`--text-mini/meta/small/body/title/h2/h1`(仅尺寸)、`--lh-tight 1.4 / snug 1.55 / body 1.7`、`--card-lift`(微阴影)、`--card-spacing: --spacing(4)=16`、`.fa-figure`(衬线·tabular·-0.01em)、`--fa-figure-size: 1.875rem`。**但 `--text-*` 未注册进 `@theme`,只有 `.md-content` 在用,UI 组件全绕开写 raw `text-xs/sm`。**

设计语言两条主线:**字阶 = 重要性分层;间距阶 = 关系分层(间距越大关系越远)。** 二者皆为 token,一处定义、全局生效,亦为后续 dev 调试页的两组旋钮。

---

## 一、字阶(Type tiers)

每层固定 **字号 · 字重 · 行距 · 字间距**,注册进 `@theme` 生成"一个类带齐四轴"的工具类(Tailwind v4)。

| 工具类 | 字号 | 字重 | 行距 | 字间距 | 用途 |
|---|---|---|---|---|---|
| `text-figure` | 1.875rem / 30 | 500(衬线·tabular) | 1.05 | −0.01em | 大金额 / KPI 数字 |
| `text-display` **新增** | 1.5rem / 24 | 600 | 1.15 | −0.015em | 空状态主问句 / 页面级大标题(补 20→30 空档) |
| `text-h1` | 1.25rem / 20 | 600 | 1.25 | −0.01em | 主区块标题 / md h1 |
| `text-h2` | 1.125rem / 18 | 600 | 1.30 | −0.005em | 次级标题 / md h2 |
| `text-title` | 1rem / 16 | **600**(原 500) | 1.35 | 0 | **卡片标题**(造层级关键招) |
| `text-body` | 0.875rem / 14 | 400 | 1.6 | 0 | 正文 / 气泡 / md 段落 |
| `text-small` | 0.8125rem / 13 | 500 | 1.45 | 0 | 强调小字 / 链接 / 表格 / 代码 |
| `text-meta` | 0.75rem / 12 | 400 | 1.45 | 0 | 次要 UI / 副标题(原 `text-xs` 主力归此) |
| `text-caption` | 0.6875rem / 11 | 500 | 1.40 | +0.02em | 计数 / 时间戳 / 标签 |
| `text-overline`(可选·签名) | 0.6875rem / 11 | 600 | 1.30 | +0.08em · `uppercase` | 分组小标,做"被设计过"的标志元素 |

**注册方式(Tailwind v4 `@theme`)**——每层连同子属性:

```css
@theme {
  --text-title: 1rem;
  --text-title--line-height: 1.35;
  --text-title--font-weight: 600;
  --text-title--letter-spacing: 0em;
  /* …其余各层同构… */
}
```

- **收编 `--lh-*`**:行距并入各层 `--text-x--line-height`;`.md-content` 段落继续用 `text-body`(其 1.6 取代散用的 `--lh-body 1.7`,如需更松可单点微调)。删除/弃用零散 `leading-*`、`tracking-*`(层已自带)。
- **`.fa-figure` → `text-figure`**:把衬线·tabular·字间距收进 figure 层;`--fa-figure-size` 退役或别名到 `--text-figure`。
- **字重对比**:正文 400、强调/标题 500–600、标题 600——取代"medium 一统"。

## 二、间距阶(Spacing scale + 语义角色)

**干净刻度(4px 网格,砍掉 6/10 半步)**:`2 / 4 / 8 / 12 / 16 / 24 / 32 / 48`(对应 Tailwind `gap/p`: `0.5 / 1 / 2 / 3 / 4 / 6 / 8 / 12`)。

**语义角色(按"关系强度"取值,统一使用)**:

| 角色 | 值(px) | Tailwind | 关系强度 |
|---|---|---|---|
| 图标↔文字 / inline | 8 | `gap-2` | 同一元素内 |
| 列表项之间(组内) | 4 | `gap-1` | 强关联,同组 |
| 卡内堆叠 | 12 | `gap-3` | 同卡不同行 |
| 卡片内边距 | 16 | `p-4` | 卡的呼吸(沿用 `--card-spacing`) |
| 卡片 / 块之间 | 16–24 | `gap-4`~`gap-6` | 并列块 |
| 分组 / 职能区之间 | 24–32 | `gap-6`~`gap-8` | **现在缺的档** |
| 大区分隔 | 48 | `gap-12` | 强切断 |
| 页面外框 | 24–32 | `p-6`~`p-8` | 整体框白 |

**原则**:间距越大=关系越远;非对称留白把分组标题"绑"到它领的组上(上多下少);列表用一致行高 + 4px 组内间距形成节奏。

> v1 用 Tailwind 数字 `gap/p` 类按上表角色落地即可;若后续要给 dev 页暴露"间距旋钮",再把宏观档(section/region)提成 `--space-*` 语义 token。

## 三、迁移映射(current → token)

**字号**:

| 现状 | → | 目标 |
|---|---|---|
| `text-xs`(12,×206) | → | `text-meta`(多数);极小计数/时间戳 → `text-caption` |
| `text-sm`(14,×94) | → | `text-body`(正文);密集次要 → `text-small` |
| `text-base`(16,×9) | → | `text-body` 或 `text-title`(按语境) |
| `text-lg` / `text-xl`(各 1) | → | `text-h1` / `text-display` |
| `text-[10px]` / `[0.625rem]` / `[11px]` | → | `text-caption` |
| `text-[13px]` | → | `text-small` |
| `text-[length:var(--fa-figure-size)]` + `.fa-figure` | → | `text-figure` |
| CardTitle `text-sm font-medium` | → | `text-title`(16/600) |
| Card 正文 `text-xs/relaxed` | → | `text-meta` |
| 散用 `font-medium`/`leading-*`/`tracking-*` | → | 删除(层自带);仅保留刻意覆盖 |

**间距**:

| 现状 | → | 目标 |
|---|---|---|
| `gap-1.5`(6,×65) | → | `gap-2`(8);确属极紧 → `gap-1`(4) |
| `gap-2.5`(10,×16) | → | `gap-2`(8) 或 `gap-3`(12) |
| 卡组 / 章节之间(现无) | → | 补 `gap-6`/`gap-8`(24/32) |
| 卡片内边距不一 | → | 统一 `p-4`(16) |

## 四、按界面落地(层级 / 呼吸落点)

- **总览(cockpit)**:KPI 数字 → `text-figure`;卡片标题 → `text-title`;**KPI 条 / 头条行 / 经营·税务行 / 合规细条之间用 `gap-6`(24)**拉开;卡内 `gap-3`、内边距 `p-4`。
- **对话空状态**:主问句"今天要处理什么?" → `text-display`(24);预制按钮文字 → `text-body`;与输入框间距已修(`mb-8`)。
- **侧栏(app-nav)**:导航行统一行高(约 36–40,`px-3` + 一致 `py`/`min-h`),**组内 `gap-1`(4)、分组之间(新对话/总览/资料 ↔ 最近)`gap-4`~`gap-6`**;会话列表项一致节奏。
- **知识库 / 文件 / 设置 / md 答案**:字阶 + 间距阶随系统迁移(字号/间距统一);**注:徽章/状态"配色"按先前决定暂不动**,本 spec 只统一其字号与间距,不改其颜色。

---

## 五、实施步骤(增量、可截图验)

- **Phase 0 — 定义 token**:在 `globals.css @theme` 写入字阶 10 层(含子属性)+ 收编 `--lh-*`;确认间距刻度(砍半步的是规约,不需 token)。此阶段**零视觉变化**(token 尚未被消费)。
- **Phase 1 — shadcn 原语**:`card.tsx`(CardTitle→`text-title`、CardDescription→`text-meta`)、`button`、`input` 等基础件采用层。高杠杆(下游自动继承)。
- **Phase 2 — 总览**:标题/数字/宏观间距落地 → 截图验。
- **Phase 3 — 对话**:主问句/正文 → 截图验。
- **Phase 4 — 侧栏**:行高节奏 + 分组间距(套用间距语言)→ 截图验。
- **Phase 5 — 知识库/文件/配置/其余**:字阶 + 间距迁移(不动徽章配色)。
- **Phase 6 — 收尾**:grep 扫残留 `text-xs/sm`、`gap-1.5/2.5`、`leading-*`、`tracking-*`;把 `.md-content` 完全并入统一字阶(它已用 `--text-body` 等,基本对齐)。

## 六、验证

- 每阶:`npm run typecheck` + `npm run lint`(改我的文件)+ **截图对比**该界面(已有 Playwright 截图管线:注入首启跳过标志 + mock 数据)。

## 七、约束与不动项(红线)

- **纯视觉**:不碰逻辑、数据、取数、工具。
- **保持密度**:正文锁 14、小字锁 ~12;层级靠顶层(figure/display/title)+ 宏观间距,**不整体放大**(财务工具的密度是特性)。
- **不动**:`error.tsx` / `global-error.tsx` 兜底硬编码色(主题崩溃时也要可读)、`file-type-icon.tsx` 文件类型身份色——皆刻意保留。
- **徽章配色**:知识库/文件状态徽章的 **颜色** 按先前决定暂不动(本 spec 只统一其字号/间距)。
- **暗色**:字阶/间距 mode 无关(尺寸不随明暗变),只定义一次;颜色仍走既有明/暗双套。

---

## 八、调整性 & 与 dev 页的关系

迁移后每层/每角色都是单一 token,**调一档 = 改 `globals.css` 一行,全局生效**——这正是后续 dev 调试页要覆盖的变量。**先 token 化 + 迁移、再做 dev 页**:没有这一步,dev 页的"字号/字重/行距/间距旋钮"无对象可调。

---

## 九、工作量 / 风险

- 触点:字号 ~300、间距 ~150,机械、纯视觉、零逻辑。
- 风险:低(多为等尺寸换名 + 删冗余修饰);分阶推进、每阶可回退、可截图验。
- 顺序价值:Phase 0–1 一旦完成,后续每个界面都是"换类"小活,且收益(层级 + 呼吸)立现。
