# 聊天文件预览与输入交互 V4 Spec

## 范围

本次增量只覆盖以下 7 条需求：

1. 修正 `txt` 和共享文件组件的多余边框/背景问题，确保所有文件格式不再出现不必要的内容框。
2. 输入区附件去掉背景色，关闭按钮移到右上角，整体高度收窄。
3. `+` 添加附件按钮 hover 需要明确背景反馈。
4. 点击 `+` 后，先在按钮正上方弹出仅包含“添加照片”“添加文件”的小窗，再触发系统文件选择。
5. 左侧 `置顶` / `最近` 分组按钮不显示边框和背景。
6. 当前选中对话行去掉边框、减弱背景并收窄高度。
7. 右上角面板按钮与右侧栏按钮的尺寸、圆角、背景框风格统一到左侧菜单样式。

## 用户需求拆解

### 1. 文件预览与共享文件样式

- `txt` 预览不能再使用额外的深色圆角块容器。
- 非图片附件在输入区不能再出现共享按钮样式带来的额外圆角/椭圆背景。
- 预览页里不同格式文件只保留必要的文档页/画布容器，不再额外叠加无意义边框块。

### 2. 输入区附件条

- 附件 chip 默认无底色、无额外边框。
- 关闭 `x` 固定在附件项右上角。
- 附件项整体高度更紧凑。
- 点击文件主体仍然直接进入右侧预览。

### 3. 添加附件入口

- `+` 按钮 hover 时必须有明显背景反馈。
- 点击 `+` 时不直接打开系统选择器，而是在按钮正上方弹出一个轻量菜单。
- 菜单只保留两个动作：
  - `添加照片`
  - `添加文件`
- `添加照片` 仅选择图片文件。
- `添加文件` 保持现有通用文件上传能力。

### 4. 左侧导航层级

- `置顶` / `最近` 作为分组切换器，不应表现成当前激活项。
- 当前会话行仍需有选中态，但只保留更轻的背景，不带描边。
- 当前会话行高度比之前更窄，避免视觉抢眼。

### 5. 右上角操作按钮统一

- 面板按钮与右侧栏按钮默认尺寸、圆角、内边距、hover 背景与左侧菜单按钮统一。
- 激活态仍需有背景，表示当前正在使用或已打开。

## 自动化测试

1. 预览组件：
   - `txt` 预览存在独立文本页容器，不再只渲染深色 `pre` 块。
   - spec 覆盖“添加照片 / 添加文件”菜单文案。

2. 聊天页结构：
   - 存在两个隐藏文件输入：图片选择和通用文件选择。
   - `+` 按钮切换附件菜单，而不是直接 `click()` 通用文件输入。
   - 附件菜单只包含 `添加照片`、`添加文件`。

3. 左侧导航与顶部按钮样式：
   - `置顶` / `最近` 的 active 样式不再要求边框/背景。
   - 当前对话行 active 样式不再包含边框。
   - 面板按钮、右侧栏按钮样式尺寸与左侧按钮选择器保持一致。

4. 输入区附件样式：
   - `attachment-chip-main` 会重置共享按钮遗留的圆角/最小尺寸。
   - 附件关闭按钮有绝对定位样式。

## 手工验收

1. 打开带文件的聊天页并上传 `txt/docx/xlsx`
   - 输入区附件无额外背景块
   - 关闭按钮在右上角
2. 点击 `txt` 进入右侧预览
   - 文本内容不再包在额外深色圆角框内
3. hover `+` 按钮
   - 能看到明确背景反馈
4. 点击 `+` 按钮
   - 先出现上浮菜单
   - 菜单只有“添加照片”“添加文件”
5. 打开 `/chat/recent?id=14`
   - `置顶` / `最近` 分组按钮无背景边框
   - 当前对话行只有较轻背景，无描边，行高更紧
6. 查看聊天页右上角
   - 面板按钮和右侧栏按钮的框体尺寸、圆角、风格和左侧菜单按钮一致
