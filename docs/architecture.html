Finance Agent · 系统架构

Next.js App Router + Claude Agent SDK + 本地 SQLite + Python Worker，单机部署的财务工作台。点击任意模块查看一句话职责。

前端 UI 后端 / API Agent / Harness 数据 / 存储 外部服务 容器 / 系统边界 (虚线) → 数据流 (标注协议)
Agent / Harness 设计理念：
1) Agent 负责理解、拆步骤、解释结果；确定性动作（报销/薪税/金蝶/Excel 生成）一定下沉到工具。
2) Harness 通过 system-prompt + skills + hooks + allowed-tools 给 Agent 装上世界观和护栏：路径沙箱、风险确认、AskUserQuestion 转写、耗时打点都在 hooks 链里。
3) Router 先用 Haiku 做轻量分类，再交给主模型跑完整 turn；长上下文用 session_id + resume 续接；子任务用 Subagent Runner 隔离上下文窗口。
4) 跨会话事实用 Memory Store 自动抽取 → SQLite → 下次 prompt 回灌。
5) 一切默认本机：SQLite、文件、向量索引、Python worker 都跑在 Tauri 进程里；外网出口只有 Claude API、金蝶 API 与可选的本机 WS sidecar。
