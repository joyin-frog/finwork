# 报销校验 Skill

适用流程：报销处理。

## 工作方式

1. 识别报销表字段和发票关键信息，先展示字段映射，再执行批量校验。
2. 调用 `check_reimbursement_batch` 校验（只读）：缺字段、金额异常、单笔超标、批内发票号重复，以及**与发票台账的跨月历史重复报销**。
3. 异常清单按风险排序输出（历史重复最优先），异常原因写清楚方便财务复核；不得隐藏或合并异常。
4. 财务复核并明确表示通过后，调用 `record_reimbursement_invoices` 把通过的发票登记进台账——这是下次跨月查重的依据，漏登记会导致查重失效。
5. 报销标准不确定时，用 search_knowledge 查公司报销制度，不得凭记忆回答。

## 执行要求

- 不自动确认报销通过；台账登记必须在财务确认之后。
- 不直接写入金蝶。
- 台账报告"已登记"的发票必须提示财务人工核实是否重复报销。
