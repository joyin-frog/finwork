# 税务优惠线索发现 Skill

**定位：线索发现器，非顾问。Agent 不替用户拍税务决策（红线 6）。**

---

## 执行流程

### 第一步：读画像 / 补问

1. 读系统提示 C 段中的公司画像。
2. 若画像为空或缺关键字段，向用户补问（AskUserQuestion），收集后调用 `update_company_profile` 更新（系统自动弹确认）。
3. **关键字段**：region（地区）、zones（园区）、taxpayerType（纳税人类型）、isHighTech（高新）、industry（行业）、scaleRevenueWan（年营收）。

### 第二步：匹配稳定全国性优惠（零联网，直接输出）

按画像匹配以下稳定规则，命中则输出，未命中则跳过：

**小型微利企业所得税优惠**
- 条件：年应纳税所得额 ≤ 300 万；从业人数 ≤ 300 人；资产总额 ≤ 5000 万
- 优惠：应纳税所得额 × 25% 再 × 20%（实际税率 5%）
- 依据：财税〔2023〕12 号 | https://www.chinatax.gov.cn/chinatax/n810341/n810765/c5203989/content.html
- **适用条件待你/税务师核实**

**高新技术企业所得税优惠**
- 条件：isHighTech = true（已持有或正在申请高新证书）
- 优惠：企业所得税按 15% 税率（标准 25%）
- 依据：《企业所得税法》第二十八条；认定管理办法（2016 修订）
- 官方参考：https://www.chinatax.gov.cn/chinatax/n810341/n810765/c5203989/content.html
- **适用条件待你/税务师核实**

**研发费用加计扣除（100%）**
- 条件：开展研发活动；建立研发辅助账/专账；非负面清单行业（烟草/娱乐/房地产/金融等）
- 优惠：研发费用可按 100% 比例加计扣除（即税前多扣 1 倍研发费）
- 依据：财税〔2023〕28 号 | https://www.chinatax.gov.cn/chinatax/n810341/n810765/c5234505/content.html
- **适用条件待你/税务师核实；当前比例须联网确认是否有最新调整**

**增值税小规模纳税人免税/优惠**
- 条件：taxpayerType = 小规模；月销售额 ≤ 10 万（季 30 万）
- 优惠：免征增值税（目前按国家最新政策执行）
- 依据：财税〔2023〕1 号等系列政策（须联网查最新有效期）
- **适用条件及有效期待联网确认**

**软件企业增值税即征即退**
- 条件：经认定软件企业；销售自产软件产品
- 优惠：增值税实际税负超 3% 部分即征即退
- 依据：财税〔2011〕100 号 | https://www.chinatax.gov.cn/chinatax/n810341/n810765/c101433/content.html
- **适用条件待你/税务师核实**

### 第三步：联网搜索地方性 / 最新政策

对以下场景联网搜索（用 WebSearch，优先官网域名：chinatax.gov.cn / gov.cn / 地方税局）：

1. **地方政府补贴/奖励**：按 region/zones 搜索，例如「临港新片区企业所得税优惠 site:gov.cn」
2. **园区专项政策**：有 zones 时搜园区政策（招商引资奖励/地方留存返还等）
3. **行业专项**：按 industry 搜索行业性优惠（如集成电路、高端制造等）
4. **当年最新政策变化**：搜「{年份}小微企业税收优惠」「{年份}研发加计扣除最新」等

**联网规则（红线 4 硬约束）**：
- 只引用官方源（国家税务局 / 省市税局 / 财政部 / 地方政府官网）
- 每条必须有文号（如财税〔2023〕28 号）和链接（可 WebFetch 验证页面是否真实存在）
- 明确标注「待核实」
- 搜不到官方源时，输出「未搜到可靠官方依据」，**不编造**
- 落审计（使用 WebSearch/WebFetch 属于联网/外发，须在工具调用过程中自动落 audit_logs）

### 第四步：输出格式

每条线索格式：

```
### [线索名称]
- **类型**：全国性 / 地方性 / 行业专项
- **适用条件**：……（简要）
- **优惠内容**：……
- **官方依据**：[文号] [链接]
- **⚠️ 注意**：适用条件待你/税务师核实，本清单仅供参考。
```

末尾加声明：
> 以上为线索发现，不构成税务建议。申报前请与税务师或主管税务机关确认适用性。

---

## 执行约束

- **只发现，不替用户享受/申报**（红线 6）：若用户要求"帮我直接办理"，用 AskUserQuestion 说明"申报流程需与税务师配合，我只能提供线索"。
- 搜不到官方源 = 说没有，不用"可能有""据了解"等模糊表述（红线 4）。
- 不独立给出"你一定能享受/不能享受"的结论——这是实质性税务判断，留给税务师（红线 6）。
