# 薪税计算 Skill

适用流程：薪税计算（累计预扣预缴法）。

## 工作方式

1. 读取工资模板或员工工资数据（Excel 用 Read/run_python 解析）。
2. 调用 `calculate_payroll_batch` 计算：累计预扣预缴算法在本 skill 的固定脚本 `scripts/payroll.py`（确定性、可调口径；parity 见 `selftest_payroll.py`），工具侧只管 DB 接力 / 确认门 / 审计。自动接力本年度已确认月份的累计数。**不要自己心算或用 Python 临时算个税**。
3. 本年度首次给某员工算工资时，必须向财务索要扣缴端的年初至今累计数（累计收入、累计五险一金、累计专项附加、累计已预扣税额）和本年任职月数，作为 `ytd` 传入；并提醒财务核对累计起点。
4. 把计算结果（含每人的计算过程）展示给财务核对；明细要能回答"为什么这个月扣税变多了"（税率档跳变）。
5. **财务明确表示确认后**，调用 `confirm_payroll_period` 使该月生效；确认后才会作为下月累计基础。
6. 已确认月份默认拒绝重算；只有财务明确要求重算时才传 `overwriteConfirmed: true`（会留审计日志）。

## 执行要求

- 计算结果必须引用税率配置版本；政策数字以配置为准，不得凭记忆改税率。
- 计算失败的员工会在结果里显式列出，必须转达财务人工处理，不得跳过。
- 最终工资和个税结果必须由财务确认，agent 不得自行确认。
