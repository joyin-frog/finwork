# 金蝶凭证 Skill

两条入口,后半程一致:
- **纸质单据照片**(主线):用户选一个装单据照片的文件夹 → OCR → 凭证草稿
- **已审批的报销/薪资数据**:直接进科目映射 → 草稿

## A. 单据照片 → 凭证(主线)

### 1. 扫描分组
用户选文件夹后拿到路径。调 **`scan_slip_folder`** 自动识别结构分组:**一组 = 一张凭证的全部材料**。一级子文件夹各成一组(一笔:单据+多发票+回单),根目录散文件/多页PDF 各自一组。两种组织与混用都支持——不用自己猜哪些文件是一笔。

### 2. 逐组取文本 + 提取字段
**对每一组**,组内文件逐个调 **`read_document`**(PDF/Excel/Word 提文字、图片和扫描件 PDF 自动 OCR;**不要自己 run_python 写 OCR / import pytesseract**),再把一组的多个文件**合起来当一笔**理解:
- 单据(报销单/付款单):金额、摘要、审批、**原借款栏**
- 多张发票:每张品名→一个借方科目(多发票=多借方明细);发票金额之和应对上单据合计
- 回单:付款账户/方式 → 贷方
从每组提取一张凭证的字段:
- 单据类型(费用报销单/付款申请单/…,看印刷表头,最可靠)
- 日期、收款方/报销人、摘要
- **金额线索**:各明细行数字、合计数字、大写金额文本(原样,别自己改废笔)
- **「原借款」栏 + 「应退(补)款」栏**(报销单底部):判断走普通还是预借款冲销分录的开关,必提取

手写摘要/被红章压住的金额 rapidocr 常读不准——读不清就标⚠️,不猜。必要时对单张用视觉兜底(图已在附件通道)。

### 3. 查对照表(一次查全)
`search_knowledge "金蝶科目对照表"` 取用户维护的「关键词→科目码→维度值」条目(从零沉淀,可能为空)。整理成 mappings 数组备用。

### 4. 一次批量处理(关键:别逐张逐工具调,会超时)
把**所有单据**提取的字段汇成 slips 数组,**一次调 `process_voucher_batch`**:
- 每张传:`date` / `lineItems`(费用明细,每行 summary+amountYuan) / `totalYuan` / `capitalText`(大写) / `advanceYuan`(原借款栏,有则走冲销) / `payeeName` / `departmentName`
- 传第 3 步的 `mappings`;科目表工具内部自动读
- 工具一次返回:每张的金额勾稽结果 + 科目映射 + 多行分录(含预借款冲销) + **汇总统计** + **对照清单行**

> ⚠️ **不要**对每张单据分别调 check_voucher_amount / map_voucher_account / build_voucher_lines——那是几十次 LLM 往返会超时。用 process_voucher_batch 一次搞定。单张零散核对时才用那些细粒度工具。
> 餐饮费默认记差旅费,仅明确宴请客户才记业务招待费(6602.06)。
> 若结果带 notice(在用示例表):先让用户 `import_kingdee_accounts` 导入真表再继续。

### 5. 汇总确认(不逐张打断)
展示 batch 返回的汇总大表(✅自动 N / ⚠️待确认 M / ❌失败)。用户挑 ⚠️ 行。
对待确认项(勾稽不平、科目未命中),用 **一个 `AskUserQuestion` 一次带多题**(面板左右切换),科目给 options 点选、金额让用户核对。确认后只对这些行补齐(改 slips 里对应字段,重跑 batch 或局部用细粒度工具)。

### 6. 沉淀对照表
用户确认科目后,把「关键词→科目码→维度值」追加进对照表(知识库文件),**明示写了什么**:如「已记入:杰强→6602.24 劳务费→综合部」。下次自动命中。

### 7. 出清单
batch 已返回对照清单行(`sheet`:表头+行,列对齐金蝶录入界面,借贷分列)。用 `run_python`(openpyxl)把它写成 .xlsx 存 output_dir。金蝶云星空无导入入口,这是「对照手填清单」,用户照着敲。列顺序由工具保证,别自己拼。

## B. 报销/薪资数据 → 凭证
已有审批后的结构化数据时,跳过 1–2,直接从第 3 步起走。

## 硬性要求
- 科目编码必须经 `map_voucher_account`/`query_kingdee_accounts` 验证存在,不凭记忆编造
- 金额一律走 `check_voucher_amount` 勾稽,不心算大写、不自行认定被遮数字
- 借贷必须平衡(totalDebit === totalCredit)
- 换公司提示:重导科目表 + 换对照表文件 + 改公司名,避免旧科目码串味

## Mock 注意
当前金蝶为模拟模式:草稿带 "simulated": true,不写真实金蝶;真实推送需配置金蝶 OpenAPI 密钥。科目表未导入时回落示例表(query/校验带 notice),示例码非真实仅演示。
