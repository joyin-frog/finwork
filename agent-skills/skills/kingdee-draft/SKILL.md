# 金蝶凭证 Skill

两条入口,后半程一致:
- **纸质单据照片**(主线):用户选一个装单据照片的文件夹 → OCR → 凭证草稿
- **已审批的报销/薪资数据**:直接进科目映射 → 草稿

## A. 单据照片 → 凭证(主线)

### 1. 取图
用户选文件夹后拿到路径。用 `run_python` 列出目录下所有图片(.jpg/.png/.jpeg/.webp,按文件名排序;跳过隐藏文件)。

### 2. 逐张 OCR + 提取字段
对每张图 `run_python` 调 `workers/finance_worker.py ocr-image <路径>`(rapidocr,已开 use_angle_cls 自动摆正横拍)。从 OCR 文本提取:
- 单据类型(费用报销单/付款申请单/…,看印刷表头,最可靠)
- 日期、收款方/报销人、摘要
- **金额线索**:各明细行数字、合计数字、大写金额文本(原样,别自己改废笔)
- **「原借款」栏 + 「应退(补)款」栏**(报销单底部):判断走普通还是预借款冲销分录的开关,必提取

手写摘要/被红章压住的金额 rapidocr 常读不准——读不清就标⚠️,不猜。必要时对单张用视觉兜底(图已在附件通道)。

### 3. 金额勾稽(不心算)
调 `check_voucher_amount`,传明细行(元)/合计(元)/大写文本。工具内做大写解析 + 三来源比对:
- ok=true → 金额可信,自动用
- ok=false(不平/单来源/大写糊)→ 标⚠️,列候选值,交用户确认。**绝不自己判定哪个数对**

### 4. 科目映射
- 先 `search_knowledge "金蝶科目对照表 <收款方/费用类型>"` 查对照表(用户维护,从零沉淀)
- 调 `map_voucher_account`,传摘要文本 + 查到的对照表条目。工具会:命中→科目码经科目表验证存在才输出、维度类型(部门/供应商…)自动带出;未命中/编码失效→明确告知,不编造
- 未命中的:据摘要 + `query_kingdee_accounts` 提建议,但**必须让用户确认**,不硬猜三级明细

> 若 query/map 返回带 notice(在用示例表):先让用户给金蝶导出的科目表、`import_kingdee_accounts` 导入真表再继续,否则科目码导不进他们金蝶。

### 5. 汇总确认(不逐张打断)
全部识别完,调 `summarize_vouchers` 出汇总大表(✅自动 N / ⚠️待确认 M / ❌失败)。展示大表,用户挑⚠️行。

对待确认项,用 **一个 `AskUserQuestion` 一次带多题**(面板左右切换逐题答),别一题一弹。科目类给 options 点选,金额类让用户核对。

### 6. 沉淀对照表
用户确认科目后,把「关键词→科目码→维度值」追加进对照表(知识库文件),并**明示写了什么**:如「已记入对照表:杰强→6602.24 劳务费→综合部」。下次同关键词自动命中。

### 7. 构造多行分录(含预借款冲销)
一张单据 = 一张凭证,但凭证是**多行独立借贷**(不是一借一贷)。用 `build_voucher_lines`:
- 传各费用明细行(每行一科目+部门维度) + 付款科目
- **看第 2 步提取的「原借款」栏**:有金额就传 `advanceYuan` + `payeeName`,工具自动生成冲销分录(贷 其他应收款-个人往来 1221.03 挂报销人、差额进银行);栏空就走普通(贷银行合计)
- 工具返回的 `balanced` 必须为 true;不平衡别往下走
- 餐饮费默认记差旅费,仅明确宴请客户才记业务招待费(6602.06)

### 8. 出草稿 + 校验 + 清单
- `export_kingdee_draft` 生成凭证草稿 → `validate_kingdee_voucher` 校验借贷平衡/科目有效
- 金蝶云星空**没有 Excel 导入入口**,所以最终产物是「对照手填清单」:用 xlsx skill / `run_python` 生成一张 Excel,列对齐金蝶凭证录入界面(摘要/科目编码/核算维度/借方/贷方),用户照着往金蝶敲。不是导入模板,是省去翻单据、判科目的时间

## B. 报销/薪资数据 → 凭证
已有审批后的结构化数据时,跳过 1–3,直接从第 4 步(科目映射)起走。

## 硬性要求
- 科目编码必须经 `map_voucher_account`/`query_kingdee_accounts` 验证存在,不凭记忆编造
- 金额一律走 `check_voucher_amount` 勾稽,不心算大写、不自行认定被遮数字
- 借贷必须平衡(totalDebit === totalCredit)
- 换公司提示:重导科目表 + 换对照表文件 + 改公司名,避免旧科目码串味

## Mock 注意
当前金蝶为模拟模式:草稿带 "simulated": true,不写真实金蝶;真实推送需配置金蝶 OpenAPI 密钥。科目表未导入时回落示例表(query/校验带 notice),示例码非真实仅演示。
