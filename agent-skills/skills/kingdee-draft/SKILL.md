# 金蝶凭证 Skill

## 适用流程
kingdee - 将审批后的报销/薪资数据导出为金蝶 K/3 Cloud 记账凭证草稿

## 目标
1. 从用户提供的报销单据/薪资计算结果提取会计科目映射
2. 在生成凭证前，先查知识库确认科目编码（调用 search_knowledge "金蝶科目 报销科目映射"）
3. 回忆用户之前的导出偏好（长期记忆已自动注入对话上下文，直接参考即可）
4. 调用 query_kingdee_accounts 查科目表，确认借方/贷方科目存在。**若返回带 notice(说明在用示例表),先让用户给金蝶导出的科目表、用 import_kingdee_accounts 导入贵司真表再继续**——否则草稿里的科目码导不进他们的金蝶
5. 调用 export_kingdee_draft 生成凭证草稿
6. 调用 validate_kingdee_voucher 校验借贷平衡和科目有效性
7. 对校验失败的项目给出修正建议，修正后重新导出

## 执行要求
- 必须查知识库确认科目映射规则——不可凭记忆编造科目编码
- 必须先查用户偏好：公司名、常用科目、凭证格式偏好
- 必须校验借贷平衡（totalDebit === totalCredit）
- 凭证摘要应包含：报销人/薪资对象、费用类型、期间
- 生成后展示草稿 JSON 并以表格形式呈现
- 提醒用户：这是模拟模式，正式凭证需在配置金蝶 API 密钥后推送到真实系统
- IMPORTANT: 任何科目编码必须在 query_kingdee_accounts 返回的列表中

## Mock 注意事项
当前金蝶为 v0.1 模拟模式：
- 科目表:优先用贵司经 `import_kingdee_accounts` 导入的真表;未导入时回落示例表(query/校验会带 notice 提示),示例科目码/余额非真实、仅供演示
- 凭证不会写入真实金蝶系统
- 导出结果带 "simulated": true 标记
- 如果用户要求真实推送，说明需要配置金蝶 OpenAPI 密钥
