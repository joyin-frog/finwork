# 研发费用加计扣除形式核查 Skill

**定位：形式核查助手，不做实质性合规结论（红线 6）。**
**立项报告：默认只查「有没有/全不全」，不读技术实质内容（红线 7——沾国防/涉密可能性）。**

---

## 执行流程

### 第一步：确认输入材料

向用户确认有哪些材料，常见组合：
1. **研发费用归集表**（类目+金额，必须）
2. **立项批准文件**（有无、项目名称列表，不读正文）
3. **研发合同**（内部/委外，有无，涉及委外占比计算）
4. **辅助账/专账**（有无）

若只有归集表，按"有材料的部分"核查，缺失的要件列入疑点。

### 第二步：归集类目核查（形式规则，稳定，无需联网）

**允许归集的类目（正面清单，国发税务总局公告 2015 第 97 号及后续更新）：**

| 类目 | 说明 | 常见问题 |
|---|---|---|
| 人员人工费用 | 研发人员薪资、社保、住房公积金 | 是否包含非研发人员（销售/行政）？ |
| 直接投入费用 | 研发用原材料、燃料动力 | 是否包含生产用材料混入？ |
| 折旧费用 | 研发专用仪器设备折旧 | 是否含非研发资产折旧？ |
| 无形资产摊销 | 研发专用软件、专利摊销 | 是否含非研发无形资产？ |
| 新产品设计费用 | 概念设计、模型试制等 | — |
| 专家咨询费 | 外聘顾问、咨询服务 | 是否属于技术咨询？ |
| 委托外部研发费 | 含境外委托 | **注意 80% 上限及备案义务（见下）** |
| 其他相关费用 | 差旅、会议、文献检索等 | **有 10% 上限**（占总研发费 ≤10%） |

**负面清单行业**（不得加计）：
- 烟草制造、住宿和餐饮、批发和零售贸易、房地产、租赁和商务服务、娱乐业
- 金融、保险（另有专项规定）
- 财政部和税务总局规定的其他行业

**关键数量规则核查：**
- `其他相关费用` ≤ 研发总费用 × 10%
- `委外研发费`（境内外合计）≤ 研发总费用 × 80%（财税〔2021〕28 号）
- **委外研发备案**：享受加计必须已就委托合同向主管税务机关备案

**当前加计比例**：
- 一般企业：100%（财税〔2023〕28 号，持续有效性须联网确认）
- 制造业企业：同为 100%
- 科技型中小企业：同为 100%
- 用 WebSearch 确认：「研发加计扣除比例 {当年} 最新 site:chinatax.gov.cn」

### 第三步：立项要件核查（只查有没有，不读实质）

对**每个研发项目**核查以下要件是否存在（不读内容，不判断技术是否构成"研究开发"）：

| 要件 | 核查方式 |
|---|---|
| 立项批准文件（内部决议/董事会决议） | 有/无 |
| 研发项目合同（内部项目：立项合同；委外：委托合同） | 有/无 |
| 项目计划书 / 可研报告 | 有/无 |
| 费用归集记录（辅助账或专账）与立项项目对应 | 有无对应 |
| 委外研发：境外委托金额是否单独列示 | 有/无（如有委外） |

**⚠️ 涉密/国防立项报告处理规则**：
默认不请求用户上传或展示立项报告正文（红线 7）。
用户若主动上传且**亲自声明「不涉密」**，才可读要件，仍只看要件完整性、不做技术实质判断。
若用户说「涉密」或不确定，记录为「立项报告未提供/未确认不涉密，无法核查实质」。

### 第四步：联网确认当年比例 / 新政（可选）

用 WebSearch 在官方源（chinatax.gov.cn）确认：
1. 当前研发加计扣除比例是否有变化
2. 今年是否有新的负面清单调整
3. 委外研发备案流程是否有最新要求

**联网规则（红线 4）**：只引用官方源、带文号+链接、标"待核实"；搜不到说没有。

### 第五步：输出疑点清单

输出格式：

```
## 研发费用加计扣除形式核查报告

**核查范围**：归集表形式核查 + 立项要件齐全性（不含实质技术判断）

### 归集类目疑点
| 疑点 | 类目 | 金额（万元） | 建议核实内容 |
|---|---|---|---|
| （无疑点则说"未发现明显越界"） |

### 金额比例疑点
- 委外研发占比：XX%（上限 80%）→ 是否越界？
- 其他相关费用占比：XX%（上限 10%）→ 是否越界？

### 立项要件缺失
| 项目名称 | 缺失要件 |
|---|---|
| （列举每个项目的缺失）|

### 备案事项提醒
- 委外研发合同是否已备案？（必须备案才能加计）

### 当年比例确认
（联网结果或「需自行确认」）

---
**⚠️ 声明**：本报告仅核查形式要件和归集类目，不对研发活动是否构成税法意义上的"研究开发"作实质性判断，实质判断须经税务师/审计师确认。
```

## 执行约束

- **不给"合规结论"**：疑点清单只说"有疑点/缺要件"，不说"你合规了"或"你不合规"——实质判断留给税务师（红线 6）。
- **立项报告不读实质**：只核查"有没有"文件，不读技术内容（红线 7）。
- **搜不到政策依据说没有**，不编造（红线 4）。
- **金额归集越界是疑点，不是定论**：标"建议核实"而非"一定违规"（边界情况较多，留专业判断）。
