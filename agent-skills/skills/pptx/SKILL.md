## 设计创意

财务汇报默认结论先行：先放核心判断、口径和关键数字，再放支撑过程。缺少来源的数据一律标待补或追问，绝不编数。

**不要制作无聊的幻灯片。** 白底上的简单项目符号不会给人留下深刻印象。为每一张幻灯片考虑以下列表中的创意。

### 开始之前

- **选择一个大胆的、与内容相呼应的色彩方案**：调色板应当让人觉得是为这个主题专门设计的。如果把你的颜色换到一个完全不同的演示文稿中仍然"能用"，那说明你的选择还不够有针对性。
- **主导色优先于均等**：一种颜色应占主导地位（60-70% 的视觉比重），辅以 1-2 个辅助色调和一个鲜明的强调色。切勿让所有颜色比重均等。
- **深色/浅色对比**：标题页和总结页使用深色背景，内容页使用浅色背景（"三明治"结构）。或者全程使用深色背景以营造高级感。
- **坚持一个视觉主题**：选择一个独特的元素并重复使用——圆角图片框、彩色圆圈中的图标、单侧粗边框等。在每一张幻灯片中都贯彻它。

### 色彩方案

选择与主题匹配的颜色——不要默认使用通用的蓝色。以下调色板可作为灵感：

| 主题 | 主色 | 辅助色 | 强调色 |
|-------|---------|-----------|--------|
| **午夜行政风** | `1E2761`（海军蓝） | `CADCFC`（冰蓝） | `FFFFFF`（白色） |
| **森林与苔藓** | `2C5F2D`（森林绿） | `97BC62`（苔藓绿） | `F5F5F5`（奶油色） |
| **珊瑚活力** | `F96167`（珊瑚红） | `F9E795`（金色） | `2F3C7E`（海军蓝） |
| **温暖陶土** | `B85042`（陶土红） | `E7E8D1`（沙色） | `A7BEAE`（鼠尾草绿） |
| **海洋渐变** | `065A82`（深蓝） | `1C7293`（蓝绿） | `21295C`（午夜蓝） |
| **炭灰极简** | `36454F`（炭灰） | `F2F2F2`（米白） | `212121`（黑色） |
| **蓝绿信赖** | `028090`（蓝绿） | `00A896`（海沫绿） | `02C39A`（薄荷绿） |

### 为每张幻灯片

**每张幻灯片都需要一个视觉元素**——图片、图表、图标或形状。纯文本幻灯片很容易被遗忘。

**布局选项：**
- 双栏（左文右图）
- 图标 + 文本行（彩色圆圈中的图标，加粗标题，下方为描述）
- 2x2 或 2x3 网格（一侧为图片，另一侧为内容块网格）
- 半出血图片（左侧或右侧全幅）并叠加内容

**数据展示：**
- 大型统计数据突出显示（大号数字 60-72pt，下方为小标签）
- 对比列（前后对比、优缺点、并排选项）
- 时间线或流程图（带编号的步骤、箭头）

**视觉润色：**
- 章节标题旁的小彩色圆圈中的图标
- 用于关键统计数据或标语的斜体强调文本

### 字体排版

**选择有趣的字体搭配**——不要默认使用 Arial。选择有特色的标题字体，并与干净的正文字体搭配。

| 标题字体 | 正文字体 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 字号 |
|---------|------|
| 幻灯片标题 | 36-44pt 加粗 |
| 章节标题 | 20-24pt 加粗 |
| 正文 | 14-16pt |
| 说明文字 | 10-12pt 淡化 |

### 间距

- 最小 0.5 英寸的页边距
- 内容块之间 0.3-0.5 英寸的间距
- 留出呼吸空间——不要填满每一寸地方

### 避免（常见错误）

- **不要重复相同的布局**——在各幻灯片之间变化使用分栏、卡片和突出显示
- **不要居中对齐正文**——段落和列表应左对齐；仅标题居中
- **不要吝啬大小对比**——标题需要 36pt 以上，才能与 14-16pt 的正文区分开
- **不要默认使用蓝色**——选择能反映特定主题的颜色
- **不要随意混用间距**——选择 0.3 英寸或 0.5 英寸的间距并始终如一地使用
- **不要只设计一张幻灯片而让其余幻灯片保持朴素**——要么完全投入设计，要么始终保持简洁
- **不要创建纯文本幻灯片**——添加图片、图标、图表或视觉元素；避免简单的标题 + 项目符号
- **不要忘记文本框内边距**——当将线条或形状与文本边缘对齐时，将文本框的 `margin` 设置为 0，或偏移形状以考虑内边距
- **不要使用低对比度元素**——图标和文本都需要与背景有强烈的对比；避免浅色文本放在浅色背景上，或深色文本放在深色背景上
- **绝对不要在标题下使用强调线**——这是 AI 生成幻灯片的标志；应使用留白或背景色来代替

## QA（必需）

**假设存在问题。你的工作就是找出它们。**

你第一次渲染的结果几乎从不正确。将 QA 视为一次缺陷查找，而不是确认步骤。如果你第一次检查就发现零个问题，那说明你看得还不够仔细。

### 内容 QA

```bash
python -m markitdown output.pptx
```

检查缺失的内容、拼写错误、顺序错误。

**使用模板时，检查是否残留占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 返回了结果，在宣布成功之前先修复它们。

### 视觉 QA

**⚠️ 使用子代理**——即使只有 2-3 张幻灯片也要用。你一直盯着代码，会看到你期望的东西，而不是实际存在的东西。子代理拥有全新的视角。

将幻灯片转换为图片（参见 [转换为图片](#转换为图片)），然后使用以下提示：

```
目视检查这些幻灯片。假设存在问题——找出它们。

寻找：
- 重叠元素（文字穿过形状、线条穿过文字、堆叠的元素）
- 文本溢出或在边缘/框边界处被截断
- 为单行文本定位的装饰线，但标题换行成了两行
- 来源引用或页脚与上方内容冲突
- 元素间距过近（< 0.3 英寸）或卡片/部分几乎相碰
- 不均匀的间隙（一处有很大空白区域，另一处却很拥挤）
- 距幻灯片边缘的边距不足（< 0.5 英寸）
- 列或类似元素未对齐一致
- 低对比度文本（例如，浅灰色文本在奶油色背景上）
- 低对比度图标（例如，深色图标在深色背景上没有对比鲜明的圆圈）
- 文本框过窄导致不必要的换行
- 残留的占位符内容

对于每张幻灯片，列出问题或需要关注的区域，即使很轻微。

阅读并分析这些图片：
1. /path/to/slide-01.jpg（预期：[简要描述]）
2. /path/to/slide-02.jpg（预期：[简要描述]）

报告所有发现的问题，包括轻微的。
```

### 验证循环

1. 生成幻灯片 → 转换为图片 → 检查
2. **列出发现的问题**（如果没发现问题，请更挑剔地再看一次）
3. 修复问题
4. **重新验证受影响的幻灯片**——修复一个问题常常会引出另一个问题
5. 重复，直到一次完整的检查没有发现新问题

**在完成至少一次修复并验证的循环之前，不要宣布成功。**

## 转换为图片

将演示文稿转换为单独的幻灯片图片以进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这会创建 `slide-01.jpg`、`slide-02.jpg` 等文件。

要在修复后重新渲染特定的幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 依赖项

- `markitdown[pptx]` - 文本提取
- `Pillow` / `defusedxml` - 缩略图网格
- `pptxgenjs` - 从零开始创建
- LibreOffice（`soffice`）- PDF 转换（通过 `scripts/office/soffice.py` 为沙箱环境自动配置）
- Poppler（`pdftoppm`）- PDF 转图片
