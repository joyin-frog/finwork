# 经营分析 Skill v2（四能力 × 三基准）

适用场景：用户上传或提供资产负债表、利润表（或金蝶/会小企导出的多 sheet Excel），需要生成经营分析表、计算财务比率、与同比/预算对比。

---

## 执行流程（先聊后生成）

### 第一步：对话确认（红线 4/6）

**不要立即生成报告**。先与用户确认：

1. 数据截止日（asOf，如"2025-12-31"）和结算状态（草稿/已确认）
2. 文件单位（元 or 万元）
3. 是否有同比基准（上年同期数/上期资产负债表期初数）
4. 是否有预算数（预算表单位通常为万元，需归一到元后传入）
5. 有无 T2 下钻需求（事业部/料工费/期间费用明细）

如缺关键信息，用 `AskUserQuestion` 收集，不要估算或填空（红线 4）。

---

### 第二步：读取 Excel 文件（T1/T2/T3 分层）

**严格按三层分类处理**：

#### T1 标准三表（用固定脚本确定性解析，**别自己写解析代码**）
- 用 `run_python` 跑本 skill 自带的**固定脚本**（不是每次现写 openpyxl）：
  ```
  python3 agent-skills/skills/business-analysis/scripts/parse_statements.py <财报.xlsx>
  ```
- 脚本**确定性**地:探测「行次 / 期末余额 / 本年累计金额 / 上年同期」列位(兼容各家金蝶列位漂移)、按行次取资产负债表与利润表关键科目、按科目名兜底取合计与现金流净额、判单位;**只读三大表 sheet,天然跳过 T3 客户实名明细(红线 7)**;输出 canonical JSON(利润表含 `prior` 上年同期作同比基准)。
- **禁止自己写 openpyxl 解析代码**:同样输入每次现写结果可能不同、不可复现(违红线 2)。脚本输出字段直接喂 `generate_business_analysis`。

**单位处理**:脚本输出含 `unit` 字段。若为「万元」,把所有数值 ×10000 归一为「元」再传工具(红线 2/3)。

#### T2 下钻层（确认式处理）
- 用户自定义费用表、事业部归集表、产品利润表等
- 先询问用户确认映射关系，再用 `run_python` 确定性归集
- 事业部维度从公司画像读取（`readCompanyProfile().revenueDimensions`），无则先问用户

#### T3 敏感层（**绝对禁止进入上下文**，红线 7）
- `应收账款明细`、`应收/到账期清单`、`进项税额明细` 等含客户实名的 sheet
- **发现即跳过，不读入、不提取、不传给工具**
- 告知用户：「该 sheet 含客户实名敏感数据，本次分析已跳过」

---

### 第三步：调用工具生成分析（红线 2）

所有确定性财务比率**必须调用 `generate_business_analysis` 工具计算**，**禁止在自然语言回答中心算或估算**。LLM 心算不可复现、不可回算。

**工具参数准备清单（传入前核对）**：

| 参数 | 必填 | 说明 |
|---|---|---|
| `balanceSheet` | 是 | 会小企01行次取数，单位归一到元；`prior` 填期初数 |
| `incomeStatement` | 是 | 会小企02行次取数，`prior` 填上年同期列 |
| `budget` | 否 | 预算表数据，**万元×10000 归一到元再传** |
| `asOf` | 是 | 数据截止日 |
| `status` | 是 | 草稿/已确认/已锁定 |

**调用前向用户复述提取的关键科目数，确认无误再调用**。

---

### 第四步：展示报告与解读

- 展示 Markdown 三基准分析表（四能力 × 本期/同比/预算列）
- 无基准数据时说明「无基准」，不补估算
- 只给分析依据和可选方向，**不替用户拍经营决策**（红线 6）
- 如需决策用 `AskUserQuestion` 交回用户

---

### 第五步（可选）：生成 Excel / Word 报告

确认后用 `Skill` 加载 `xlsx` / `docx` 技能，用 `run_python` 生成：
- Excel：四能力×三基准明细表，带口径列
- Word：经营分析报告（含数据信任链脚注）

## 核心红线

| 红线 | 本 skill 落点 |
|---|---|
| **2 数值正确性** | 比率/增长率/ROE/杜邦全走工具，禁止心算；入参声明单位（元） |
| **3 数据信任链** | 工具自动带 asOf + status；跨期聚合先对齐口径 |
| **4 "我不知道"** | 无基准→「无基准」；查不到→如实说；不编近似数 |
| **6 决策边界** | 只给分析结果，不替老板拍板；要决策用 AskUserQuestion |
| **7 合规驻留** | T3 客户明细 sheet 不读入、不传、不进上下文 |

## 典型问答

**Q：用户说"帮我看看这个财报"（上传会小企 Excel）**
A：先确认截止日、单位、是否有预算，说明 T3 sheet 会跳过；用 run_python 分层读取；调 generate_business_analysis；展示四能力表并解读关键指标。

**Q：用户问"净利润增长了多少"**
A：让用户提供财报文件或报出关键科目数；调工具算同比增长率；不直接在回答里心算。

**Q：用户要"生成经营分析 Excel"**
A：先对话确认数据（第一步），再调工具生成分析，最后加载 xlsx skill 产出 Excel。
